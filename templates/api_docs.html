Документация Public API

Как получить токен, настроить ключ и вызывать endpoint вакансий: GET /api/public/vacancies (и /api/public/vacancies/semantic-search)

Что делает Public API
JobEco анализирует вакансии из Telegram (LLM + embeddings + дедуп) и хранит структуру в БД.
Public API отдаёт готовые записи наружу по секретному ключу.
1) Как получить токен
Открой раздел ключей: /api/keys.
Нажми Create, задай фильтры/лимиты и создай ключ. Секретный токен показывается один раз — сохрани его.
В запросах токен передаётся в заголовке:
X-API-Key: <token>
2) Настройка ключа (фильтры и лимиты)
Фильтры применяются на уровне ключа и работают для всех запросов с этим токеном.
domains — OR по доменам (web3/ai/igaming/tech/gaming/traffic)
role — подстрока через ILIKE
recruiter — подстрока через ILIKE
company_domain — точное совпадение
location_type — remote/hybrid/office
risk_label — high-risk
salary_min_usd, salary_max_usd
Лимиты: requests_per_minute и daily_quota. При превышении возвращается HTTP 429.
3) Публичный endpoint
URL:
GET /api/public/vacancies
Query-параметры:
page — номер страницы (default: 1)
per_page — размер страницы (default: 50, max: 200)
include_blocks / include_raw_text: сейчас игнорируются, потому что blocks и raw_text возвращаются всегда.
3.1) Публичный vector/semantic endpoint
URL:
GET /api/public/vacancies/semantic-search
Query-параметры:
q — строка запроса для embedding
page — номер страницы (default: 1)
per_page — размер страницы (default: 20, max: 200)
Выдача сортируется по близости embedding (cosine distance). Фильтры по ключу применяются так же, как и для /api/public/vacancies.
4) Пример запроса
curl -s \\
  -H "Accept: application/json" \\
  -H "X-API-Key: YOUR_TOKEN" \\
  "http://YOUR_HOST/api/public/vacancies?page=1&per_page=20"
5) Формат ответа
Ответ всегда вида: { page, per_page, total, items[] }.
Внутри items для каждого элемента есть:
id, title, company_name
role, domains, risk_label
summary (summary_en), skills (alias: stack)
contacts — объект типов/значений (например: {"Telegram":"@user"})
source_url — ссылка на оригинальный пост (best-effort)
Блоки вакансии: description, responsibilities, requirements, conditions, raw_text
semantic_similarity — только для semantic-search

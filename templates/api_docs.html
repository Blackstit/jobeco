Public API Documentation

Endpoints: GET /api/public/vacancies and GET /api/public/vacancies/semantic-search

Authentication
Create an API key at /api/keys. The secret token is shown once — save it.
Pass in header:
X-API-Key: <token>
GET /api/public/vacancies
Paginated list of vacancies with full data.
Query parameters:
page — page number (default: 1)
per_page — page size (default: 50, max: 200)
search — text search in title, company, raw_text
domains — filter by domains, repeatable (e.g. domains=web3&domains=ai)
location_type — remote / hybrid / office
seniority — trainee / junior / middle / senior / lead / head / c-level
employment_type — full-time / part-time / project / freelance / internship
salary_min_usd, salary_max_usd — salary range filter (USD)
score_min, score_max — AI quality score filter (0-10)
risk_label — "high-risk" or "not-high-risk"
company_name — search by company name (substring)
GET /api/public/vacancies/semantic-search
Vector similarity search. Same filters + response as above.
Additional parameters:
q — search query (required). Results sorted by embedding similarity.
All filters from /api/public/vacancies also work here. Response includes semantic_similarity field (0.0–1.0).
Example request
curl -s \
  -H "Accept: application/json" \
  -H "X-API-Key: YOUR_TOKEN" \
  "https://YOUR_HOST/api/public/vacancies?page=1&per_page=10&domains=web3&seniority=senior&score_min=5"
Response format
{"page": 1, "per_page": 10, "total": 42, "items": [...]}
Each item contains:
id, title, company_name, role
domains — array of domain tags (web3, ai, igaming, etc.)
seniority — trainee/junior/middle/senior/lead/head/c-level
employment_type — full-time/part-time/project/freelance/internship
english_level, language_requirements — e.g. {"english": "B2", "russian": "C1"}
location_type, salary_min_usd, salary_max_usd
risk_label — null or "high-risk"
summary, skills (alias: stack)
contacts — typed object, e.g. {"Telegram": "@user", "Application Form": "https://..."}
source_url, created_at
scoring — AI quality assessment:

  "total_score": 7.7,
  "overall_summary": "Well-defined vacancy with clear compensation...",
  "red_flags": [],
  "scoring_results": [
    {"criterion": "Tasks & KPI clarity", "key": "tasks_and_kpi", "score": 8, "weight": 0.3, "summary": "..."},
    {"criterion": "Compensation clarity", "key": "compensation_clarity", "score": 9, "weight": 0.25, "summary": "..."},
    {"criterion": "Stack & processes", "key": "tech_stack_and_ops", "score": 6, "weight": 0.2, "summary": "..."},
    {"criterion": "Requirement logic", "key": "requirement_logic", "score": 7, "weight": 0.15, "summary": "..."},
    {"criterion": "Company profile", "key": "company_profile", "score": 8, "weight": 0.1, "summary": "..."}
  ]
}
company — enriched company profile (from Perplexity):
{
  "name": "Maple Finance",
  "website": "https://maple.finance",

  "industry": "DeFi",
  "size": "50-200",
  "founded": "2019",
  "headquarters": "Melbourne, Australia",
  "summary": "Onchain asset manager providing institutional-grade lending...",
  "socials": {"twitter": "https://twitter.com/maboroshi", "linkedin": "..."},
  "domains": ["web3", "fintech"]
}
Vacancy blocks: description, responsibilities, requirements, conditions, raw_text
semantic_similarity — only for semantic-search (0.0–1.0)
Key-level filters
Filters can also be set at the API key level (in config). Query parameters override key-level config when provided.
domains, location_type, risk_label
salary_min_usd, salary_max_usd
role, recruiter — substring match (ILIKE)
Rate limits: requests_per_minute and daily_quota. Returns HTTP 429 when exceeded.
